# V2 修仙电子宠物硬件清单

本文用于规划全新一代修仙电子宠物原型机。新设备不沿用 RP2040-Zero 硬件约束，目标是支持更高清的动画、触摸与实体按键、Wi-Fi、Bluetooth LE、电池供电、语音采集与播放，以及通过服务端接入 LLM。

本文是原型阶段采购与验收依据，不是最终量产 BOM。第一阶段先使用高度集成的开发板验证玩法、动画、语音、网络和续航，验证完成后再决定是否设计定制 PCB。

## 1. 设计目标

- 便携式电池供电，可通过 USB-C 充电和烧录。
- 480×480 高清彩色屏幕，支持明显的角色动画、粒子和页面过场。
- 支持电容触摸和四个实体游戏按键。
- 支持 2.4GHz Wi-Fi 和 Bluetooth LE。
- 支持麦克风输入、扬声器输出和按住说话。
- 支持 IMU 动作交互、RTC 时间事件和 TF 卡资源扩展。
- LLM 在云端或自有服务端运行，设备不保存模型服务商 API 密钥。
- 网络中断时，本地游戏、基础台词、动画和存档仍可正常工作。
- 不使用震动马达，反馈由画面、声音和实体按键共同完成。

## 2. 推荐主板

首选主板：**Waveshare ESP32-S3-Touch-AMOLED-2.16，带 3.7V 1000mAh 电池版本**。

主要规格：

| 项目 | 规格 |
|---|---|
| 主控 | ESP32-S3R8，双核最高 240MHz |
| 内存 | 8MB PSRAM |
| Flash | 16MB NOR Flash |
| 屏幕 | 2.16 英寸 AMOLED，480×480，电容触摸 |
| 无线 | 2.4GHz Wi-Fi、Bluetooth LE 5 |
| 音频 | ES8311 播放、ES7210 双麦克风采集与回声消除 |
| 传感器 | QMI8658 六轴 IMU |
| 时间 | PCF85063 RTC |
| 存储 | TF 卡槽 |
| 电源 | AXP2101 电源管理、USB-C、3.7V 锂电池充放电 |
| 板载按键 | GPIO18 用户键、PWR 键、BOOT 键 |

选择该主板的原因：

- AMOLED 的黑场、对比度和响应速度适合修仙法术、灵气、雷劫和夜间洞府场景。
- QSPI 屏幕比占用大量并行 GPIO 的 RGB 屏更便于集成音频和传感器。
- 板载双麦克风、音频编解码和回声消除，适合后续开发 LLM 语音互动。
- 已包含 IMU、RTC、TF 卡和电池管理，可减少首版飞线和模块数量。
- 支持 ESP-IDF 和 Arduino，但正式项目优先采用 ESP-IDF，以获得更好的任务调度、网络、音频和图形控制。

- [硬件与引脚文档](https://docs.waveshare.com/ESP32-S3-Touch-AMOLED-2.16)

### 电池采购注意事项

- 原型阶段优先购买官方带电池版本，不单独寻找所谓“同规格”电池。
- 主板使用 3.7V 单节锂电池和 MX1.25 2Pin 接口，但同规格插头不保证极性一致。
- 未核对正负极前禁止插入第三方电池。
- 禁止使用鼓包、破损、无保护板或来源不明的锂电池。
- 外壳必须为电池预留固定、绝缘、散热和轻微膨胀空间。

## 4. 实体按键扩展

目标是保留四个游戏按键，用于战斗、炼丹、节奏修炼和盲操作。触摸屏负责页面导航、抚摸和复杂选择，板载 GPIO18 用户键优先作为“按住说话”键。

主板的大部分 GPIO 已被屏幕、TF 卡、音频、触摸、RTC、IMU 和 USB 使用。四按键不应在未核对原理图的情况下直接占用剩余 GPIO，推荐通过共享 I2C 总线接入 GPIO 扩展器。

| 部件 | 最低数量 | 推荐数量 | 规格与要求 | 用途 |
|---|---:|---:|---|---|
| GPIO 扩展板 | 1 | 2 | MCP23017，确认支持 3.3V | 扩展 K1–K4 和后续控制信号 |
| 轻触按键 | 4 | 8–10 | 侧按或正按型号按外壳结构选择 | 四个游戏输入 |
| 按键帽 | 4 | 1 套 | 与轻触按键及外壳匹配 | 改善操作手感 |
| 100nF 电容 | 4 | 10 | 陶瓷电容，可选 | 硬件辅助去抖 |
| 10kΩ 电阻 | 4 | 10 | 可选；优先评估扩展器内部上拉 | 外部上拉和调试 |

推荐按键定义：

| 按键 | 默认功能 |
|---|---|
| K1 | 确认、攻击、冒险 |
| K2 | 上移、技能一、调高火候 |
| K3 | 下移、技能二、调低火候 |
| K4 | 返回、防御、放弃 |
| 板载 GPIO18 | 按住说话、打断语音播放 |

接线原则：

- MCP23017 与主板共享 GPIO14=SCL、GPIO15=SDA。
- 接入前必须运行 I2C 地址扫描，确认地址不与触摸、AXP2101、QMI8658、RTC、ES8311 和 ES7210 冲突。
- 按键采用低电平有效，一端接扩展器输入，另一端接 GND。
- 软件必须实现去抖、短按、长按和组合键识别。
- GPIO0 与启动模式相关，不作为普通游戏按键使用。
- GPIO19、GPIO20 为原生 USB，不占作普通 GPIO。
- GPIO43、GPIO44 默认保留给 UART 调试，原型初期不复用。

## 5. 音频方案

主板已经集成语音采集和播放链路，第一阶段不额外购买独立麦克风、I2S 功放或语音识别模块。

音频交互方式：

1. 玩家按住 GPIO18 用户键开始录音。
2. 屏幕播放灵气聚集动画。
3. 松开按键后结束录音并上传到自有服务端。
4. 服务端完成语音识别、LLM 推理和语音合成。
5. 设备流式播放语音，同时显示字幕和口型动画。
6. 网络不可用时切换到本地预置台词和音效。

扬声器采购前需要确认：

- 主板实际出货包是否已包含扬声器。
- 扬声器额定阻抗和功率。
- 插头型号、针距和正负极线序。
- 扬声器尺寸是否能装入计划中的音腔。

外壳应为扬声器设置独立音腔，麦克风孔与扬声器出声孔应尽量分开，避免回声和啸叫。麦克风孔只覆盖薄型防尘网，不使用厚泡棉完全封闭。

## 8. 明确排除的部件

当前方案不使用以下部件：

- 震动马达及其 MOSFET、续流二极管和驱动电路。
- 独立语音识别模块。
- 摄像头。
- GPS 和 4G 模块。
- 外置 Wi-Fi 天线。
- 独立树莓派或本地大模型主机。
- ESP32-P4 双芯片方案。
- 3000mAh 以上的大型锂电池。
- 第一阶段定制 PCB。

不使用震动马达后，受击、暴击、突破和通知反馈由以下方式承担：

- AMOLED 闪白、屏幕位移、残影和粒子。
- 扬声器音效与角色语音。
- 动画关键帧停顿和界面颜色变化。
- 实体按键操作反馈。

## 9. 采购顺序

1. 购买主板、电池、扬声器、TF 卡、USB-C 数据线和 USB 功率计。
2. 验证屏幕、触摸、Wi-Fi、Bluetooth LE、TF 卡、RTC、IMU、录音、播放、电池和充电。
3. 运行图形压力测试，确认 480×480 动画帧率和内存余量。
4. 运行语音链路测试，确认录音质量、扬声器音量和回声情况。
5. 扫描 I2C 地址并核对原理图。
6. 购买 MCP23017、四按键及装配材料。
7. 制作按键原型板并完成软件去抖。
8. 测量真实功耗、温升和续航。
9. 根据实物尺寸和声学测试结果设计第一版外壳。
10. 游戏原型验证完成后，再决定是否设计定制 PCB。

## 10. 首轮硬件验收清单

- [ ] USB-C 可稳定烧录和输出日志。
- [ ] AMOLED 无坏点，触摸定位正常。
- [ ] 480×480 动画压力测试无明显撕裂和长期卡顿。
- [ ] PSRAM、Flash 和 TF 卡容量读取正确。
- [ ] Wi-Fi 能稳定连接并通过 TLS 访问自有服务端。
- [ ] Bluetooth LE 能用于首次配网或近距离发现。
- [ ] 双麦克风可稳定采集语音。
- [ ] 扬声器无明显破音、底噪和啸叫。
- [ ] RTC 断网后仍能维持设备时间逻辑。
- [ ] IMU 能识别抬起、倾斜、轻摇和静止。
- [ ] 电池电量、充电状态、休眠和唤醒正常。
- [ ] 连续高亮、Wi-Fi和语音工作时没有异常温升。
- [ ] K1–K4 无抖动、误触或丢按键。
- [ ] 断网时本地游戏、基础台词、动画和存档仍可使用。
- [ ] LLM API 密钥仅保存在服务端，不写入固件和设备文件。

## 11. 后续定制 PCB 的决策条件

只有原型完成以下验证后才进入定制 PCB：

- 核心玩法已经证明需要四按键、触摸、语音和 IMU。
- 目标动画能在 ESP32-S3 上达到可接受帧率。
- 1000mAh 电池的真实续航数据已经获得。
- 扬声器音腔和麦克风开孔方案已经稳定。
- Wi-Fi、BLE 和 LLM 服务端协议已经稳定。
- 已确定最终外壳尺寸、按键位置、屏幕方向和重心。
- 已确认 16MB Flash、8MB PSRAM 与 TF 卡的资源分配方案。

若 ESP32-S3 无法满足最终动画目标，再评估 ESP32-P4 加无线协处理器的高性能方案，不在首版原型中提前引入该复杂度。
