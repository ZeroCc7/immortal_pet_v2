# 首页实现进度

最后更新：2026-07-23

## 已完成

- 首页改为全屏修仙场景背景；日间与夜间背景均已转为 PNG 并纳入默认资源包，避免目标配置缺少 JPEG 解码器而不显示背景。
- 左上角已保留境界、等级与经验进度的位置；中间不再显示旧版圆形“宠物表情”。
- 未配置小智时，不再展示默认聊天气泡、配网提示或无意义的系统文字；仅在玩家主动交互或后续 AI 返回内容时显示气泡。
- 底部入口改为四张大尺寸竖版玉绿金边卡片，采用“图标在上、文字在下”的布局，参考已确认的设计稿。
- 已生成并压缩四个首页图标：修炼、游历、收获、札记。图标与背景位于 `firmware/device/main/assets/immortal_pet/home/`，并已接入固件资源打包。
- 当前入口与旧版功能绑定保持兼容：修炼、游历、收获、札记；这样可先完成首页视觉验收，不改动既有动作逻辑。

## 当前限制

- 尚未编译或烧录；由 VS Code 端执行完整构建和刷写后，在真机确认资源分区与布局效果。
- 主角人物资产暂不加入首页，中心区域由场景背景承载。
- 入口尚未切换为最终的“早/中/晚：修炼、历练、休息”三时段规则；收获与札记仍复用当前原型的交互。
- 装备、商店、境界突破、角色阶段外观和本地存档尚未接入首页。

## 下一步

1. 在真机验证首页：背景是否满屏、顶部经验条是否在左上、四个按钮的触控区域与文字清晰度是否符合预期。
2. 用 `game_core` 实现并测试三时段行动：修炼、历练、休息，以及同日重复收益递减和错过时段自动休息。
3. 将四个首页入口从旧原型行为替换为最终玩法入口；商店与状态页承载装备购买、属性和突破门槛。
4. 接入等级、修为、灵石、武器/衣服/鞋子，支持逐件购买、混搭和同级三件套的纯视觉特效。
5. 增加每十级境界与角色阶段资源；小智配置完成后，仅以修仙角色口吻消费只读状态并提供对话或叙事。

## 真机验收清单

- 首页背景覆盖 480×480 显示区域，且不被容器底色遮挡。
- 左上显示境界、等级和经验条；不出现旧版表情或未配置小智的提示。
- 四张底部卡片均可辨识并可点击：修炼、游历、收获、札记。
- 点击后不出现乱码、重叠文本或异常气泡。
