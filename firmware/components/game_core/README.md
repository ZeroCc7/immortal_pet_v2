# game_core

`game_core` 是不依赖网络、LLM、LVGL 和具体板卡的确定性游戏规则组件。

当前已实现：

- 版本化 `GameState`。
- 精力上限与每 5 分钟恢复 1 点的时间推进。
- 30 秒吐纳：消耗 10 精力，完成后获得修为并改善心境。
- 后山游历：支持 10、30、60 分钟，完成后确定性结算修为、灵石、材料和羁绊。
- 活动忙碌、资源不足、时间回退、尚未完成等错误状态。

设计边界：

- 所有奖励由本地代码计算，AI 不能直接修改数值。
- 时间参数使用 Unix 秒，由 RTC/系统时间适配层提供。
- 组件当前只保存纯状态；NVS A/B 存档将在独立 storage 适配层实现。
- UI 和小智语音只调用公开接口，不直接修改 `GameState`。

ESP-IDF 工程可以把本目录作为普通 component 引入。核心接口位于：

```text
include/game_core/game_engine.h
```
